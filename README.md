# Users Registration Analysis – SQL Case Study

## 📌 Objective

Analyze a simulated user registration database to extract business insights
using SQL aggregation, filtering and date-based analysis.

---

## 🗄 Database Design

A simple `users` table including:

- Unique email constraint
- Country information
- Registration timestamp

---

## 🔎 Key Analytical Questions

1. How many users are registered in total?
2. Which country has the highest number of users?
3. Which countries have only one registered user?
4. What is the monthly registration trend?
5. What is the distribution of email domains?

---

## 📊 Insights Extracted

- Identified the country with the highest user concentration.
- Detected rare countries with only one registered user.
- Analyzed growth trend by month.
- Observed dominant email domains.

---

## 🧠 Skills Demonstrated

- GROUP BY & HAVING
- Aggregation functions (COUNT, MAX)
- Subqueries
- Date functions (DATE_TRUNC)
- String manipulation (SPLIT_PART)
- Data integrity via UNIQUE constraint
